# D3 v1 — 구조 보존형 리드 (Structure-Preserving Lead)

작성일: 2026-07-16 · 상태: 승인됨(구현 대기) · 분류: Phase 2 AI 강화 / D3 롤아웃·리드 정책

## 배경 / 목표

- Phase 2 "다음 프론티어" 조사(5중 워크플로 종합) 결과, 값싼 강도 레버는 소진됐고 봇은 이미 통계적으로 강하며 공개 티츄 SOTA 이상(폭탄 포함 풀룰). 남은 값싼 강도 레버 중 **규율(격리 bench)에 맞는 유일한 것 = D3**.
- 현재 `AiAgent.DecideLead`는 손패 5장 초과에서 사실상 **"점수 없는 가장 낮은 수"** 하나만 리드한다 (`MoveOrder.Lowest(noPoint)`, `Assets/_Project/GameFlow/Agents/AiAgent.cs:249–254`). 손패 분해·구조 인식·나가기 계획이 없다.
- 결과 약점: 가장 낮은 싱글이 스트레이트(예 2-3-4-5-6)나 풀하우스의 구성원이면, 그 카드를 홀로 리드해 **가치 구조를 깬다**.
- 공유 `AiAgent`이므로 이 변경은 **PIMC 롤아웃 리프 정책 + 라이브 리드 + Easy 직접결정** 모두에 전파된다(의도됨). 롤아웃 EV 품질 상승이 목적.

## 범위

**포함**
- `DecideLead`의 "5장 초과 · near-out 락아웃 아님" 분기에 구조 보존 필터 적용.
- 커밋 판정: 그리디 분해로 길이 ≥3 가치 콤보(스트레이트 ≥5, 트리플, 풀하우스, 연속페어)에 속한 카드를 표시.
- 구조를 깨는 싱글을 후보에서 제외한 뒤 `MoveOrder.Lowest`. 제외 후 남는 후보가 없으면 현행 그대로 폴백.
- Unity(`Assets/_Project/GameFlow/Agents/AiAgent.cs`) + core(`core/src/Tichu.GameFlow/Agents/AiAgent.cs`) 두 사본 미러 동기화.

**제외 (이번 증분 아님)**
- 완전 나가기 플래너(손패를 최소 플레이 수로 분해·순서화).
- 팔로우 경로 분해, 페어(2장) 보존, 명시적 제어(재진입) 카운팅.

## 설계

### 핵심 규칙 (한 문장)
리드 후보(현행 `noPoint`, 없으면 `pool`)에서 **가치 구조를 깨는 싱글을 제외**한 뒤 그중 가장 낮은 수를 낸다. 제외 후 남는 게 없으면 현행 그대로.

이 한 규칙이 세 효과를 동시에 낸다:
- **구조 보존** — 낮은 싱글이 스트레이트/풀하우스 구성원이면 안 깨고, 구조 밖 낮은 싱글을 대신 리드.
- **낮은 콤보 셰딩** — 낮은 카드가 전부 콤보에 묶여 헐거운 저싱글이 없으면, 싱글로 쪼개는 대신 낮은 콤보를 통째로 리드(`Lowest`가 자연히 콤보 선택).
- **제어 보존** — 5장 초과에선 `Lowest`가 이미 약한 수를 골라 고카드를 남긴다. **별도 코드 없음**(단순함 유지). 이 구간의 실질적 새 내용은 "구조 보존"이다.

### 커밋 판정 알고리즘 (보수적·그리디)
`CommittedCards(hand)` → 홀로 리드하면 구조를 깨는 카드(랭크) 집합:
1. **스트레이트(≥5)** — 기존 `StraightRanks` 재사용: ≥5 연속 랭크에 속한 카드.
2. **트리플/풀하우스** — 같은 랭크 3장 이상 → 그 랭크 카드 커밋. (최소 버전: 트리플만으로 커밋 표시; 순수 페어는 비커밋.)
3. **연속페어(stairs, ≥2쌍)** — 같은 랭크 2장씩 연속 → 커밋.

원칙: **애매하면 커밋 아님**(보수적). 최적 분해가 아니라 "명백히 깨면 안 되는 것"만 막는다.

### 변경 지점
`AiAgent.cs:249–254`의 `noPoint` 분기 한 곳. 새 헬퍼(`StructurePreservingLeads` 또는 기존 `CheapestNonStructural`/`BreaksStraight` 확장)로 구현.
- 선택 후 near-out 싱글 스왑(259–264)은 그대로 뒤에서 실행 — 콤보 선택 시 `Cards.Count==1` 아니므로 영향 없음. 불변.
- lockCombo 분기(242), ≤5장 끝내기(`MostShedding`, 237), 폴로우/폭탄/교환/티츄 전부 불변.

1. **변경 전** core `AiAgent` → `OldAiAgent.cs` sed 재생성(베이스라인 동결).
2. Unity + core 두 `AiAgent.cs`에 동일 변경.

4. **수용 기준**: 회귀(분명한 음수)면 폐기. 회귀 없으면 채택. 양수면 승리(① 선례 +34.3/R, Wilson LB 0.557).
5. EditMode(Unity) + core dotnet 스위트 그린. 옛-동작 잠금 테스트는 **의도된 변경만** 갱신(기존 `WastefulComboOvertake` 발산은 알려진 것).

## 리스크 / 불확실성
- 낮은 스트레이트를 통째로 리드하면 셰딩은 크나 상대가 받아 선을 뺏길 수 있음 → **이득/무효/손해가 진짜 불확실**. bench가 판정하고, wash면 플래너로 확장하지 않고 **파킹**.
- 그리디 분해는 비최적(트리플을 풀하우스로 쓸지 등 모호) → 보수적 커밋으로 완화.
- 공유 `AiAgent` 전파 = 격리 불가 부분 존재하나, HeuristicStrengthBench는 순수 휴리스틱 맞대결이라 이 변경의 순효과를 격리 측정.

## 테스트 계획 (TDD, 먼저 실패시킬 것)
- 스트레이트 보유 시: 그 구성 저싱글 대신 **구조 밖 저싱글**을 리드.
- 헐거운 저싱글 없음(저카드 전부 콤보): **저콤보 통째** 리드.
- 트리플 보유 시: 트리플 구성 저싱글 대신 구조 밖 싱글 리드.
- 구조 없음: **현행과 동일**(회귀 없음 보장).
- ≤5장 끝내기 경로 불변(`MostShedding`).
- near-out 락아웃 경로 불변.
