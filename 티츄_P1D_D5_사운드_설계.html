D5 — 사운드: IAudioService + AudioBank + 인게임 SFX 훅

D4의 IPlayAnimator 시임을 1:1 미러링해 IAudioService(SFX 단일 훅)를 만들고, 별도 리액터 클래스 없이 RuntimeTableView.Subscribe()의 기존 vm 관측(Played/RoundResult/SeatCall)에 한 겹 더 얹는다 — 애니와 "동일 소스". 폭탄/콜 분류는 순수 정적 SfxMap으로 격리해 EditMode 전수 검증한다. AudioBank 미할당이면 무음(no-op 3중 가드)이라 사운드 에셋 0개여도 컴파일·런타임·테스트 통과. 진실/연출 분리로 오라클 비트동일성은 그대로 보존한다.

0 D5 목표 · 한눈에

기획 잔여 아키텍처 §6.5의 사운드 묶음을 구현한다. D5 코어 = 인게임 SFX 훅 + no-op 가드. 손맛의 대부분(카드·폭탄·패스·티츄 콜·라운드 종료)은 이미 화면에 흐르는 vm 관측만으로 도출된다.

① 시임 미러링 진실/연출 분리

IAudioService.PlaySfx(SfxId) + NoOpAudioService 1파일 2형 — IPlayAnimator 컨벤션 정확 복제. 실 구현 UnityAudioService는 DoTweenPlayAnimator가 트윈만 다루듯 AudioSource만 다룬다.

② 동일 소스 훅 리액터 미신설

IPlayAnimator가 별도 리액터 없이 _anim 인라인 호출로 끝나는 것과 똑같이, Subscribe()의 Played/RoundResult/SeatCall 구독에 _audio 한 겹을 더 얹는다.

③ 순수 매핑 격리 EditMode

폭탄/콜 분류를 Subscribe 람다에 묻지 않고 SfxMap.For(GameAction) / ForCall(TichuCall) 순수 정적 함수로 뺀다 → UnityEngine 무의존, 전수 단위테스트.

이 설계는 멀티에이전트 설계 패널(최소-수술 / 테스트우선 / 모바일성능 3안) → 5축 심사 → 단일 종합 → 적대적 검증 4렌즈(오라클·no-op·수술적/YAGNI·EditMode 커버)로 도출했다. 베이스=최소-수술 골격, 여기에 테스트우선안의 SfxMap 순수 매핑을 이식하고, 모바일성능안의 보이스 풀 N=6 근거만 차용했다. 검증이 적출한 minor 11건은 모두 본 설계에 반영됐다(§1).

1 적대적 검증이 잡은 것

핵심 안전 주장은 코드 사실로 전부 입증(반증 실패)됐고, critical/major 0건. 대신 minor 결함·축소 여지 11건을 구현 전에 적출해 설계에 선반영했다.

입증된 안전(반증 실패): ⑴ 오라클/헤드리스는 AsyncGameDriver.RunRoundAsync(onApply=null) 경로라 vm을 건드리지 않음 → vm 구독에만 얹힌 오디오 훅이 코드패스에 물리적으로 부재(비트동일성 무영향). ⑵ 헤드리스 Simulator는 vm 무참조 → 10만판 무결성 그대로 유효. ⑶ 신규 asmdef 0(audio 모듈 자동 참조)·RoundBootstrap 0줄·additive 훅. ⑷ AudioReactor 미신설이 기존 IPlayAnimator 패턴과 대칭.

구현 전에 선반영한 minor 수정 (검증 4렌즈 종합)

  • S1 · SfxMap.For null/빈 Cards 가드 — GameAction.Cards는 nullable. FormatAction(L691)처럼 Kind 우선 라우팅 + Play 분기에서만 Cards != null && Count > 0 검사 후 Recognize. (NRE 차단)
  • S2 · _audio를 ctor에서 즉시 NoOp 초기화 — _anim = anim ?? NoOp과 대칭. Bind에서만 폴백하면 ctor~Bind 사이 null-window가 생긴다. → 어떤 시점에도 _audio != null.
  • S3 · UnityAudioService bank+clip 이중 가드 — PlaySfx 첫 줄에 if(_bank==null) return; clip=_bank.Clip(id); if(clip==null) return; (명시 주입으로 bank=null이 와도 무예외).
  • S4 · RoundEnd SFX는 RenderResult의 r!=null 가드 안에 — RoundBootstrap.cs:71이 매 라운드 RoundResult.Value=null로 능동 재대입한다. 무가드 구독은 라운드마다 오발 → 가드 안에 두고 "리셋 null→발화 0"을 테스트로 못박음.
  • S5 · TichuCall SFX는 전이 가드 안에 — UpdateCallBadge의 call != None && call != prev 분기 안에서만 발화(전이당 1회, None 진입 자체 차단).
  • S6 · Default_ctor 무예외 테스트는 3 소스 모두 푸시 — Played(Play/Pass) + RoundResult + SeatCall까지 밀어야 NoOp 폴백 안전을 실제로 증명.
  • S7 · AudioBankTests 인스턴스 분리 + AreSame — Clip의 _map 지연빌드 stale 함정 회피. 빈뱅크/등록뱅크를 별도 CreateInstance로, 등록→non-null은 참조동일성(AreSame)으로 단언.

2 아키텍처 · 데이터 흐름

진실(ApplySnapshot/RecordPlay)은 즉시 vm을 갱신하고, 그 vm 관측을 화면(애니)과 사운드가 같은 소스로 구독한다. 오디오는 어디에도 끼어들지 않는 순수 additive 소비자다.

vm 관측을 애니(_anim)와 사운드(_audio)가 동일 소스로 구독한다. 오라클 경로는 vm을 발화시키지 않으므로 오디오가 닿지 않는다.

2.1 배선 — RoundBootstrap 0줄, RuntimeTableView self-new 폴백

오디오 조립은 RuntimeTableView.Bind() 내부의 self-new로 처리해 DI/생성자 표면을 0 증가시킨다 — CardView/CardSpriteAtlas의 Resources.Load 폴백(L78-79)과 동일 컨벤션이다. 테스트만 스파이를 명시 주입한다(PlayAnimatorWiringTests 패턴).

// RuntimeTableView — ctor: _anim과 대칭으로 _audio도 즉시 NoOp(널-윈도 제거, 수정 S2)
public RuntimeTableView(IPlayAnimator anim = null, IAudioService audio = null)
{
    _anim = anim ?? new NoOpPlayAnimator();
    _audio = audio ?? new NoOpAudioService();
    _audioExplicit = audio != null;   // 명시 주입(테스트 스파이)이면 Bind 폴백 생략

// Subscribe(): 기존 vm 구독에 _audio 한 겹 (애니와 동일 소스)

3.3 SfxMap — 순수 매핑(수정 S1 가드 포함)

public enum SfxId { None = 0, CardPlay, Pass, Bomb, GiveDragon, TichuCall, GrandTichuCall, RoundEnd }

public static class SfxMap
{
    // 진실 로그(이미 적용된 불변 GameAction)→SFX. Kind 우선 라우팅 — Play 분기에서만 Cards 접근.
    public static SfxId For(GameAction a)
    {
        switch (a.Kind)
        {
            case GameActionKind.Pass:       return SfxId.Pass;
            case GameActionKind.GiveDragon: return SfxId.GiveDragon;
            case GameActionKind.Play:
                if (a.Cards == null || a.Cards.Count == 0) return SfxId.CardPlay;  // S1: nullable 가드
                var c = CombinationRecognizer.Recognize(a.Cards.ToArray(), TrickContext.Lead);
                return c.IsBomb ? SfxId.Bomb : SfxId.CardPlay;
            default:                        return SfxId.None;
        }
    }

    public static SfxId ForCall(TichuCall call) => call switch
    {
        TichuCall.Tichu      => SfxId.TichuCall,
        TichuCall.GrandTichu => SfxId.GrandTichuCall,
        _                    => SfxId.None,
    };
}

3.4 AudioBank · UnityAudioService

// CardSpriteAtlas.Face 패턴 — 미할당/미등록=null=소비자 무음. _map 지연 빌드.
[CreateAssetMenu(menuName = "Tichu/Audio Bank", fileName = "AudioBank")]
public sealed class AudioBank : ScriptableObject
{
    [System.Serializable] public struct Entry { public SfxId id; public AudioClip clip; }
    [SerializeField] private Entry[] clips = new Entry[0];
    private Dictionary<SfxId, AudioClip> _map;

    public AudioClip Clip(SfxId id)
    {
        if (clips == null || clips.Length == 0) return null;
        if (_map == null) { _map = new(clips.Length); foreach (var e in clips) _map[e.id] = e.clip; }
        return _map.TryGetValue(id, out var c) ? c : null;
    }
}

// 보이스 풀 N=6: 4인 동시 연출(폭탄+차례+결과) 여유. 스틸링/동적확장 미도입(YAGNI)·D6 프로파일러 조정.
public sealed class UnityAudioService : IAudioService
{
    private const float Vol = 1f;
    private readonly AudioBank _bank;
    private readonly AudioSource[] _voices;
    private int _next;

    public UnityAudioService(AudioBank bank, int sfxVoices = 6)
    {
        _bank = bank;
        var host = new GameObject("AudioRoot") { hideFlags = HideFlags.HideAndDontSave };
        _voices = new AudioSource[Mathf.Max(1, sfxVoices)];
        for (int i = 0; i < _voices.Length; i++) _voices[i] = host.AddComponent<AudioSource>();
    }

    public void PlaySfx(SfxId id)
    {
        if (_bank == null) return;                         // S3: bank 널가드
        var clip = _bank.Clip(id);
        if (clip == null) return;                         // S3: 미할당 → 무음
        _voices[_next++ % _voices.Length].PlayOneShot(clip, Vol);
    }
}
AudioListener는 App.unity 카메라의 기존 1개를 재사용(추가 생성 0). PlayOneShot(clip, volumeScale)은 per-call 제로 할당. 짧은 SFX는 임포트 Decompress On Load로 첫 재생 디코드 힉업을 줄인다(D6 프로파일러 확인).

4 no-op 3중 가드 · 오라클 보존

4.1 no-op 3중 가드 — 사운드 에셋 0개여도 통과

  1. 뱅크 부재 — AudioBank.asset 미생성 → Resources.Load<AudioBank>("AudioBank")==null → _audio가 NoOp 유지. (CardView/Atlas null 폴백과 동일 컨벤션)
  2. 클립 미할당 — 뱅크는 있으나 SfxId 미등록 → Clip(id)==null → UnityAudioService.PlaySfx가 return으로 무음(PlayOneShot 미호출 → 예외 경로 없음).
  3. 테스트/헤드리스 — NoOpAudioService 명시 주입(빈 본문) 또는 자동 폴백 → AudioSource 미생성.
D5에서 AudioBank.asset 생성은 보류 권장. 빈 .asset이 로드되면 Resources.Load!=null→UnityAudioService 경로를 타되 clip null로 무음이라 "왜 소리 안 나지" 혼동이 생긴다. 실제 .wav 도입 시 생성하면 "뱅크 부재→NoOp" 폴백이 더 명확하다. (무예외/무음은 어느 쪽이든 보장)

4.2 오라클 보존 불변식 (이중 안전)

구조적 부재

  • 오라클/헤드리스 = AsyncGameDriver.RunRoundAsync(onApply=null) → _onApply?.Invoke 단락 → vm 미호출.
  • RoundBootstrap.RunMatchAsync만 non-null onApply(L86) 전달 = 라이브.
  • 헤드리스엔 RuntimeTableView Bind 자체가 없음 → _audio 인스턴스조차 부재.

코어 불가침 + 컴파일 가드

  • SfxMap.For가 부르는 CombinationRecognizer.Recognize는 순수 읽기(FormatAction L693이 이미 동일 호출) → GameState 변형 0.
  • 테스트는 NoOpAudioService(빈 본문).
  • IPlayAnimator XML 주석(L9-10)의 불변식을 _audio가 동일 수명으로 상속.
결론: GameDriver(동기) / AsyncGameDriver(onApply=null)의 비트동일성에 오디오는 한 비트도 닿지 않는다. 기존 Sim 10만판 무결성 테스트가 그 경로를 이미 커버하며, D5는 그 경로에 코드 추가 0이다.
