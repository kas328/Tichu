D6 — Android 빌드 · 60fps · 가독성 게이트 체크리스트

Phase 1 MVP의 완료 기준(DoD)을 닫는 게이트. 코드·EditMode로는 커버 불가한 실기기 60fps 실측과 가독성·"한 판 더" 정성 평가를 체크리스트로 정의한다. 빌드 식별자는 D6에서 정리했고(productName=Tichu·com.kas328.tichu·minSdk26), 측정용 FpsOverlay(디버그 빌드 한정)를 추가했다. 실측·플레이테스트는 사람+중저가 Android 기기로 수행한다.

1.2 측정용 FPS 오버레이

Presentation/Diagnostics/FpsOverlay — Development Build에서만 표시(RoundBootstrap이 Debug.isDebugBuild 게이트로 생성, 릴리스 no-op). 평활화는 EditMode 검증된 FpsSampler(0.5s 윈도). 색: 녹 ≥55 황 30~55 적 <30.

1.3 APK 빌드 단계 (사람)

  1. Android Build Support(SDK·NDK·JDK) 설치 확인 — Unity Hub 모듈.
  2. File → Build Settings → Platform = Android로 Switch(첫 전환은 에셋 reimport로 수분 소요).
  3. Development Build 체크(FpsOverlay·프로파일러 연결용) → Build And Run(또는 APK 빌드 후 설치).
  4. 중저가 기기에 설치(가능하면 저가·구형 1대 + 중급 1대).
이번 D6 작업에서 APK 빌드는 수행하지 않았다(로컬 Android SDK/NDK 설치 여부 미확인·범위 제외). 위 단계는 사람이 실행한다. 빌드 자동화가 필요하면 별도 요청.

2 성능 게이트 — 60fps (DoD #2)

중저가 Android에서 한 판 전 구간 60fps 유지·끊김/입력지연 체감 없음. FpsOverlay 상시 관찰 + 프로파일러(Deep 아님, GC/드로콜) 병행.

  • 딜링·교환·플레이 전 구간 FPS 오버레이 녹색(≥55) 유지, 적색 스파이크 없음
  • 카드 플레이 팝/차례 펄스/결과 팝(DoTween) 시 프레임 드랍 체감 없음
  • 손패 선택·드래그·버튼 입력 → 화면 반응 지연 체감 없음
  • 여러 라운드 연속 진행 시 GC 스파이크로 인한 끊김 없음(풀링 D3 효과 확인)
  • 프로파일러: 프레임당 GC Alloc 지속 증가 없음(라운드 전환 제외), 드로콜 안정
  • 발열/배터리 — 수 분 플레이에 과도한 발열 없음(정성)
측정 팁: Build Settings Autoconnect Profiler 또는 기기 연결 후 Window → Analysis → Profiler. SFX(D5)는 AudioBank.asset 미할당 시 무음이므로 오디오 부하는 실 클립 도입(D5.1) 후 재측정.

3 가독성 게이트 (DoD #3)

"좁은 모바일 가로 화면에 4인의 손패 수·콤보·점수·선언을 한눈에"(기획 §1.2). 팔 길이에서 중저가 기기 화면을 보며 아래를 확인한다.

내 손패 / 조작

  • 손패 카드 랭크·무늬가 또렷이 읽힘(아트·라벨)
  • 선택 하이라이트(노랑)·폭탄 글로우(빨강 테두리)가 구분됨
  • 버튼(내기/패스/스몰티츄/스킵) 터치 타깃 충분·오탭 없음
  • 교환·소원·용 양도 UI가 명확

상대 3인 / 테이블

  • 좌/우/상 좌석 손패 장수·프로필이 겹침 없이 읽힘
  • 현재 차례 좌석 이름 강조색으로 즉시 식별
  • 중앙 트릭 카드·소유자 표시가 명확
  • 티츄/큰티츄 좌석 배지 + 콜 순간 플래시 인지

정보 패널

  • 좌상단 누적 점수(우리:상대) 읽힘
  • 우상단 최근 플레이 로그(5초 페이드) 따라갈 수 있음
  • 소원(마작) 표시가 눈에 띔
  • 라운드 결과 배너(팀 합산)가 명확

레이아웃 / 손맛 보조

  • 노치/펀치홀 기기에서 SafeArea로 UI 안 가림
  • 카드 플레이/차례/결과 애니가 시선을 옳게 유도
  • SFX(실 클립 도입 후) 액션과 동기 — 카드/폭탄/콜/결과
  • 가로 1920×1080 기준 스케일이 작은 화면서도 깨지지 않음

4 손맛 게이트 — 정성 (DoD #3)

  • 한 판(목표 점수)을 크래시·멈춤 없이 완주
  • 모든 룰 인터랙션(폭탄·용·개·마작 소원·티츄/큰티츄·1-2 피니시)이 UI에서 정상 동작
  • 플레이가 "답답하지 않다"(AI 딜레이·스킵 버튼으로 템포 조절 가능)
  • "한 판 더" 하고 싶은 의향 — 내부 플레이테스트 다수 합격
D5.1 이월(이 게이트의 손맛에 영향): 버튼 클릭 SFX · BGM · 설정 볼륨 패널 · OnApplicationPause BGM 음소거. 실 사운드 에셋(AudioBank.asset) 도입도 D5.1. 가독성·60fps 통과 후 손맛을 더 끌어올리는 작업.
