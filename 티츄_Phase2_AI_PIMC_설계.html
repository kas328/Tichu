Tichu · Phase 2 · AI 고도화 · Design Spec

Phase 2 — PIMC 강 AI 설계

불완전정보 + 파트너십 트릭게임 티츄의 강 AI를 PIMC(미관측 손패를 N세계로 결정화 → 현 AiAgent 휴리스틱 롤아웃 → 수별 기댓값 투표)로 구현한다. 현 휴리스틱을 롤아웃 정책으로 재사용해 신규 코드 최소, IAgent/IDecisionAgent로 나란히 삽입해 드라이버·룰엔진 무수정, 오라클(비트동일) 불침범. 멀티에이전트 워크플로(Ground 3 + Research 2[28 학술출처])로 도출.

PIMC · 결정화 N세계 현 휴리스틱 = 롤아웃 정책 신규 asmdef 0 · 드라이버 무수정 오라클 비침범 · 결정적 시드 4티어 · anytime ≤300ms DoD = Hard > 휴리스틱봇(Wilson CI)

0 결정 · 왜 PIMC

탐색 코어 = PIMC, 완료 기준 = 기획 DoD(Hard가 현 휴리스틱봇 대비 통계적 유의 승률 우위). 흥미롭게도 연구 2건은 ISMCTS를 권고했으나, 첫 증분의 단순성과 현 휴리스틱 재사용이 PIMC를 1순위로 만든다.

PIMC 채택 근거

  • 현 AiAgent 휴리스틱을 롤아웃 디폴트 정책으로 재사용 → 신규 룰/평가 코드 최소(CLAUDE.md 수술적·단순)
  • 트릭게임은 PIMC 친화(높은 leaf-correlation·disambiguation, Long et al. 2010)
  • ISMCTS의 정보집합 노드 + ComputeHash 정규화(손패 미정렬·트릭카드 미반영)보다 구현·검증 부담 작음

약점 → 단계적 방어

  • strategy fusion(숨은 카드 아는 듯 과콜·무모폭탄): 1차 수용 → Hard 자기대국 진단 → 심하면 Expert에서 EPIMC(초반 ply 연기)/ISMCTS
  • 비균등 샘플 함정(교환·콜 정보누출): Hard부터 reach-probability 가중
  • 체감강도 ≠ 객관강도: 사람다운 동점깨기 + 인간 플레이테스트
비목표: 강화학습(RL Tichu는 amateur 그침)·CFR·슈퍼휴먼·stale core/ dotnet 부활. 현실적 상한 ≈ "준수한 인간 수준".

1 아키텍처 · 삽입

탐색 AI는 기존 추상 경계로 나란히 삽입되어 드라이버·룰엔진을 한 줄도 바꾸지 않는다. 신규 asmdef 0.

탐색 AI(보라)는 IAgent/IDecisionAgent로 기존 휴리스틱·사람과 나란히 삽입. 탐색 코어(녹)는 GameFlow.Agents 순수 C#이라 스레드풀 안전. 오라클은 휴리스틱 경로에서만 검증되어 불침범.

1.1 asmdef 배치 (신규 0)

  • 탐색 코어 → Tichu.GameFlow.Agents(noEngineReferences=true·Core만 참조 → UnityEngine 비접촉·스레드풀 안전).
  • 비동기 어댑터 PimcDecisionAgent → Tichu.Presentation(이미 UniTask·R3).
  • 벤치 러너 → EditMode 비기본 카테고리([Explicit]/Category("Bench")). 신규 asmdef 불요.
난이도 주입은 RoundBootstrap.RunMatchAsync의 IDecisionAgent[] 생성부 + GameLaunchArgs.Difficulty 추가(팩토리). ⚠️ 백그라운드 진입 전 state.Clone() 필수(ctx.State는 드라이버가 in-place 변경하는 가변 공유).

2 PIMC 탐색 흐름

무거운 탐색은 DecideTurn·DecideBomb 2곳만(나머지 7결정포인트 경량). 폭탄은 경량 게이트 통과 후에만 탐색(폭탄창 매 좌석 호출 → 예산 폭발 방지).

결정화 N세계 → 각 세계 휴리스틱 롤아웃 → ScoreRound 보상 → 수별 평균 EV 투표. 동점깨기는 MoveOrder(사람다운 수 편향).

// PimcDecisionAgent.DecideTurnAsync (의사코드)
var snap = ctx.State.Clone();                 // 가변 공유 → 백그라운드 진입 전 필수
var move = await UniTask.RunOnThreadPool(() =>
    pimc.Search(snap, seedDerived, config, deadline, ct), ct);
// 예산만료 취소 ↔ 폭탄인터럽트 취소는 별 플래그로 구분(잘못된 턴 폐기 방지)
return move;                                   // 메인스레드 복귀 후 적용

3 Determinizer · 치트가드

미관측 손패를 룰 제약 충족 분배한다. 미지풀을 상대 3좌석의 공개 Hand.Count에 정확히 일치하게 채운다(불일치 시 Apply Reject).

미지풀 = 56장 − 관측좌석 Hand − 모든 WonCards − CurrentTrick.History 카드 − 버려진 개 − 나온 마작
치트 가드 (CRITICAL). GameState는 모든 Seats[*].Hand를 노출한다. Determinizer 버그로 진짜 상대 손을 그대로 두면 상대 패를 훔쳐보는 비현실적 AI(불완전정보 위배)가 된다. 단위테스트로 강제: 관측좌석 외 Hand는 재샘플로 원본과 달라지고, 관측좌석·공개정보는 불변.
비균등 샘플 함정 (Hard+). 교환·티츄 콜이 강한 정보 누출이라 균등 분배는 희박/불가능한 손을 환각해 오평가. Hard부터 reach-probability 가중(관측 행동을 했을 확률의 곱). Normal까지는 균등 허용.

4 난이도 티어 · 결정성

단일 PimcAgent + PolicyConfig struct 주입. 차이는 정책+예산뿐(별 구현 금지). 현 AiAgent private const 4개를 외부화하되 기존 AiAgent는 보존.

티어세계수롤아웃/세계시간예산추론/협력
Easy0(OFF)—즉시현 휴리스틱+ε 블런더
Normal~4~8~80ms균등 샘플·약협력
Hard~16–20~16~250msreach-prob 가중·전협력·폭탄타이밍
Expert~24~24~300ms 캡+EPIMC·블러프

주 레버 = 시간예산·세계수(가장 단조). 숫자는 시작 추정값 → P2-C 벤치로 확정·D6 모바일 실측.

4.1 결정성 · 오라클 (2층 분리)

1층 — 오라클 불침범

  • 기존 오라클(동기==비동기 ComputeHash 5시드)은 휴리스틱 경로에서만 검증 → 탐색 AI는 미투입 → 침범 0
  • PIMC 도입 후에도 기존 테스트 그린 유지 회귀

2층 — 신규 탐색 결정성

  • "고정 노드수 모드"(시간캡 OFF)에서 결정적: 시드주입 Rng만, System.Random/DateTime/Stopwatch 금지
  • 같은 (state,seed,config) → 같은 수 + ComputeHash 재생 일치
  • 시간캡 모드는 비결정적이 정상(인게임 전용, 회귀 대상 아님)

5 특수카드 · 협력 · GC

특수카드 (별 룰구현 불요)

  • 마작 소원: LegalMoves가 이미 소원수만 남김. 과거 소원패스=음의단서
  • 용 양도: 2지선다 노드, ScoreRound가 점수귀속→팀보상 후크
  • 봉황: 봉황0/1 전수→합법수 폭증, MoveOrder 상위K 가지치기
  • 개: 결정화 풀서 제외, 선양보는 팀보상 발현
  • 폭탄 인터럽트 ⚠️: 차례밖. 1차 단순(턴좌석만)·Hard부터 충실

파트너 협력

  • 팀 보상(터미널 팀 점수차) + 파트너=자기복제 정책(상용 표준)
  • 기획 §8⑤(점수몰아주기·티츄보조·선양보)는 팀보상으로 자연 발현
  • Hard: reach-prob로 파트너 신호 해석
  • 체감강도: 사람다운 동점깨기 + 인간 플레이테스트
최대 리스크 = GC. Clone()이 노드/세계마다 대량 할당 → 모바일 GC 히칭으로 D6 60fps 위반. 완화: 세계당 1회 Clone 재사용·풀링·작게 시작(worlds 4×rollouts 8)→프로파일 후 상향. UniTask.RunOnThreadPool(메인 비차단), Task.Run(Unity SyncContext) 금지.

6 벤치마크 · DoD

⚠️ 기존 10만판 Sim이 메인스레드 점유→MCP stuck. AI-vs-AI 벤치는 비기본 카테고리([Explicit]/Category("Bench"))+반복수 캡, run_tests 클래스 필터로만, PlayMode 외부. 기본 스위트에 무거운 루프 금지.
  • 동기 대량 승률: MatchRunner.RunMatch(masterSeed, agentFactory, …)에 PimcAgent(고정노드수) 주입.
  • 사고시간: 비동기 마이크로벤치 1수당 Stopwatch(평균/p95/최악).
  • 통계: 미러드딜(좌석 스왑)로 분산↓, 팀점수차 1차 지표 + 승률 Wilson 95% CI. 100-Elo(≈64%) 게이트 ~150–200판, 운 제거엔 수천판. 대조군 풀로 자기대국 편향 방지.
  • DoD 게이트: Hard가 현 AiAgent 대비 팀점수차 평균>0 + Wilson CI 하한>0.5.
